# Contributing to Terranis

Thank you for your interest in contributing to **Terranis**! We welcome contributions from the community to help build a smarter, more sustainable urban future.

Please follow these guidelines to ensure a smooth contribution process.

---

## Code of Conduct
By participating in this project, you agree to abide by our terms. Please treat all contributors and maintainers with respect.

---

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug in the source code, please open an issue on our issue tracker. Include:
* A clear description of the problem.
* Steps to reproduce the bug.
* Expected vs. actual behavior.
* Screenshots or logs if applicable.

### 2. Suggesting Enhancements
Have an idea for a new feature or improvement? Open an issue tagged with `enhancement` and describe:
* The motivation behind the change.
* How the feature should work.
* Any alternative solutions you've considered.

### 3. Pull Requests (PRs)
1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Make your changes** following our coding standards.
3. **Test your code** to ensure no existing functionality is broken.
4. **Commit your changes** using clear and descriptive commit messages (e.g., `fix: resolve sensor data polling timeout`).
5. **Push to your fork** and open a Pull Request against the `main` branch.

---

## Development Setup

To get the project running locally for development:

1. Clone your fork:
   ```bash
   git clone https://github.com/your-username/terranis.git
   cd terranis
   ```
2. Set up the backend:
   ```bash
   cd backend
   npm install
   npm start
   ```
3. Open `frontend/index.html` in your browser or serve it via a local live-server.

---

## Style Guidelines
* Write clean, readable code with appropriate comments.
* Keep commit history clean and meaningful.
* Ensure all code adheres to standard formatting (e.g., Prettier/ESLint if configured).

Thank you for helping make Terranis better!
